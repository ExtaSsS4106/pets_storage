🐾 Pet Storage API Documentation

Система учета товаров для зоомагазина / склада

📍 Базовый URL: http://127.0.0.1:8000
🔐

Аутентификация

API использует Token Authentication. После успешного входа/регистрации вы получаете токен, который нужно передавать в заголовке каждого запроса:

Authorization: Token <your_token_key>
POST /sign_up Регистрация
Создает нового пользователя и сотрудника

📥 Request Body:

{
    "full_name": "Иван Петров",
    "email": "ivan@example.com",
    "password1": "securepass123",
    "password2": "securepass123",
    "date_of_birth": "1990-01-01",
    "phone_number": "+1234567890",
    "position_id": 1
}

📋 Параметры:

Поле Тип Обязательность Описание
full_name string ✅ Да Полное имя сотрудника
email string ✅ Да Email (уникальный)
password1 string ✅ Да Пароль
password2 string ✅ Да Подтверждение пароля
date_of_birth date ✅ Да Дата рождения (YYYY-MM-DD)
phone_number string ✅ Да Номер телефона
position_id integer ✅ Да 1 - Administrator, 2 - Warehouse_Operator

✅ Успешный ответ (201):

{
    "token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b",
    "user": {
        "full_name": "Иван Петров",
        "email": "ivan@example.com"
    }
}

❌ Ошибка (400):

{
    "password2": ["Пароли не совпадают"],
    "email": ["Сотрудник с таким email уже существует"]
}
POST /login_api Вход
Авторизация пользователя
{
    "email": "ivan@example.com",
    "password": "securepass123"
}

✅ Успешный ответ (200):

{
    "token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b",
    "user": {
        "full_name": "Иван Петров",
        "email": "ivan@example.com"
    }
}

❌ Ошибка (404):

{
    "detail": "wrong password"
}
POST /logout_api 🔒 Требуется токен
Выход из системы (удаление токена)

📤 Headers:

Authorization: Token 9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b

✅ Успешный ответ (200):

{
    "detail": "Успешный выход"
}

Поле Тип Обязательность Описание

{
    "data": "user not found"
}

❌ Ошибка доступа (403):

{
    "data": "no permission acces"
}

Коды ошибок

Код Описание
200 Успешный запрос
201 Успешное создание
400 Неверные данные запроса
401 Не авторизован (нет токена)
403 Доступ запрещен (недостаточно прав)
404 Ресурс не найден
500 Внутренняя ошибка сервера
📝

Важные примечания

  • 📦 1. Каждая запись товара = одна физическая единица
    Каждая запись в БД соответствует одной коробке, мешку и т.д. Нет поля Quantity.
  • 🔄 2. Статусы товаров:
    active - активный (на складе)
    deleted - списан/удален
  • 📋 3. Действия движений:
    adding - добавление товара
    disposal - списание товара
  • 👔 4. Должности:
    1 - Administrator (полный доступ)
    2 - Warehouse_Operator (ограниченный доступ)
  • 🔑 5. Токены:
    Токен автоматически создается при регистрации/входе и удаляется при выходе
