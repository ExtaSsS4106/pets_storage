🐾 Pet Storage API Documentation

Система учета товаров для зоомагазина / склада

📍 Базовый URL: http://127.0.0.1:8000
🔐

Аутентификация

API использует Token Authentication. После успешного входа/регистрации вы получаете токен, который нужно передавать в заголовке каждого запроса:

Authorization: Token <your_token_key>
POST /sign_up Регистрация
Создает нового пользователя и сотрудника

📥 Request Body:

{
    "full_name": "Иван Петров",
    "email": "ivan@example.com",
    "password1": "securepass123",
    "password2": "securepass123",
    "date_of_birth": "1990-01-01",
    "phone_number": "+1234567890",
    "position_id": 1
}

📋 Параметры:

Поле Тип Обязательность Описание
full_name string ✅ Да Полное имя сотрудника
email string ✅ Да Email (уникальный)
password1 string ✅ Да Пароль
password2 string ✅ Да Подтверждение пароля
date_of_birth date ✅ Да Дата рождения (YYYY-MM-DD)
phone_number string ✅ Да Номер телефона
position_id integer ✅ Да 1 - Administrator, 2 - Warehouse_Operator

✅ Успешный ответ (200):

{
    "token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b",
    "user": {
        "full_name": "Иван Петров",
        "email": "ivan@example.com",
        "position": "Administrator"
    }
}

❌ Ошибка (400):

{
    "password2": ["Пароли не совпадают"],
    "email": ["Сотрудник с таким email уже существует"]
}

Товары

exists - активный товар на складе
deleted - списанный товар
delivery - товар в поставке
GET /select_all 🔒 Требуется токен АКТУАЛЬНО
Получить список всех товаров на складе с группировкой по названию, производителю, категории, типу животного и сроку годности

✅ Успешный ответ (200):

{
    "data": {
        "status": "all_products_grouped",
        "data": [
            {
                "name": "Вискас с курицей",
                "manufacturer": "Mars",
                "category": "wet",
                "animal_type": "cats",
                "expiry_date": "2025-12-31",
                "delivery_date": "2024-03-17",
                "delivery_quantity": 0,
                "total_quantity": 10,
                "deleted_quantity": 2,
                "status": "exists"
            }
        ]
    }
}

📊 Поля: delivery_quantity, total_quantity, deleted_quantity, status

GET /select_active_with_ids 🔒 Требуется токен НОВЫЙ
Получить список активных товаров (Status='exists') с группировкой и массивами ID для списания

{
    "data": {
        "status": "active_products_with_ids",
        "grouped_data": [
            {
                "name": "Корм для кошек жидкий",
                "manufacturer": "Whiskas",
                "category": "wet",
                "animal_type": "cats",
                "expiry_date": "2026-06-30",

            }

🔍 Особенность: Каждая группа содержит массив product_ids со всеми ID товаров в этой группе. Идеально для страницы списания.

POST /add 🔒 Требуется токен СТАТУС DELIVERY
Добавить новые товары на склад (статус 'delivery' для новых поставок)
{
    "new_products": [
        {
            "name": "Вискас с курицей",
            "manufacturer": "Mars",
            "category_id": 2,
            "animal_type_id": 2,
            "expiry_date": "2025-12-31",
            "delivery_date": "2024-03-17",
            "status": "delivery"
        }
    ]
}

📋 Параметры:

Поле Тип Описание
category_idinteger1-dry, 2-wet, 3-treats, 4-hay
animal_type_idinteger1-dogs, 2-cats, 3-rodents
statusstringdelivery (для поставок)

✅ Успешный ответ (200):

{
    "data": {
        "errors_count": 0,
        "list": [],
        "status": "completed"
    }
}
POST /change_status 🔒 Требуется токен УНИВЕРСАЛЬНЫЙ
Универсальный метод для изменения статуса товаров. Автоматически создает записи в движениях.
{
    "products": [
        {
            "id": 1,
            "status": "deleted"
        },
        {
            "id": 2,
            "status": "deleted"
        }
    ]
}

📋 Параметры:

Поле Тип Описание
idintegerID товара
statusstringdelivery, exists, expired, deleted

🔄 Автоматические движения:

Изменение статуса Action в движениях
→ deleteddisposal (списание)
delivery → existsrecive (поступление)
→ expiredexpired (просрочка)

✅ Успешный ответ (200):

{
    "data": {
        "success_count": 2,
        "errors_count": 0,
        "list": [],
        "status": "completed"
    }
}
📊

Движения товаров

adding - добавление товара
disposal - списание товара
recive - поступление из поставки
expired - просрочка

GET /select_movment/<str:action> 🔒 Требуется токен ОБНОВЛЕНО
Получить движения по типу действия (adding, disposal, recive, expired)

  • /select_movment/disposal - списания
  • /select_movment/recive - поступления
  • /select_movment/expired - просрочки

{

Важные примечания

  • 📦 1. Учет товаров
    Каждая запись = одна физическая единица (коробка, мешок). Нет поля quantity.
  • 🔄 2. Статусы товаров
    • exists - активный на складе
    • deleted - списан
    • delivery - в поставке
  • 📋 3. Действия движений
    • adding - добавление
    • disposal - списание
    • recive - поступление из поставки
    • expired - просрочка
  • 👔 4. Должности
    1 - Administrator, 2 - Warehouse_Operator
  • 📊 5. Группировка товаров
    Все GET запросы возвращают сгруппированные данные с количеством
  • 🆕 6. Новый метод для списания
    /select_active_with_ids - возвращает сгруппированные активные товары с массивами ID для удобного списания
