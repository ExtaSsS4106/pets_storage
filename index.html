🐾 Pet Storage API Documentation

Система учета товаров для зоомагазина / склада

📍 Базовый URL: http://127.0.0.1:8000
🔐

Аутентификация

API использует Token Authentication. После успешного входа/регистрации вы получаете токен, который нужно передавать в заголовке каждого запроса:

Authorization: Token <your_token_key>
POST /sign_up Регистрация
Создает нового пользователя и сотрудника

📥 Request Body:

{
    "full_name": "Иван Петров",
    "email": "ivan@example.com",
    "password1": "securepass123",
    "password2": "securepass123",
    "date_of_birth": "1990-01-01",
    "phone_number": "+1234567890",
    "position_id": 1
}

📋 Параметры:

Поле Тип Обязательность Описание
full_name string ✅ Да Полное имя сотрудника
email string ✅ Да Email (уникальный)
password1 string ✅ Да Пароль
password2 string ✅ Да Подтверждение пароля
date_of_birth date ✅ Да Дата рождения (YYYY-MM-DD)
phone_number string ✅ Да Номер телефона
position_id integer ✅ Да 1 - Administrator, 2 - Warehouse_Operator

✅ Успешный ответ (200):

{
    "token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b",
    "user": {
        "full_name": "Иван Петров",
        "email": "ivan@example.com",
        "position": "Admin"
    }
}

❌ Ошибка (400):

{
    "password2": ["Пароли не совпадают"],
    "email": ["Сотрудник с таким email уже существует"]
}
POST /login Вход
Авторизация пользователя
{
    "email": "ivan@example.com",
    "password": "securepass123"
}

✅ Успешный ответ (200):

{
    "token": "9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b",
    "user": {
        "full_name": "Иван Петров",
        "email": "ivan@example.com",
        "position": "Admin"
    }
}

❌ Ошибка (404):

{
    "detail": "wrong password"
}
POST /logout 🔒 Требуется токен
Выход из системы (удаление токена)

📤 Headers:

Authorization: Token 9944b09199c62bcf9418ad846dd0e4bbdfc6ee4b

✅ Успешный ответ (200):

{
    "detail": "Успешный выход"
}

Товары

GET /select_all 🔒 Требуется токен ОБНОВЛЕНО
Получить список всех товаров на складе с группировкой по названию, производителю, категории, типу животного и сроку годности

✅ Успешный ответ (200):

{
    "data": {
        "status": "all_products_grouped",
        "data": [
            {
                "name": "Вискас с курицей",
                "manufacturer": "Mars",
                "category": "wet",
                "animal_type": "cats",
                "expiry_date": "2025-12-31",
                "delivery_date": "2024-03-17",
                "total_quantity": 10,
                "active_quantity": 8,
                "deleted_quantity": 2,
                "status": "active"
            }
        ]
    }
}

🔍 Что изменилось: Теперь товары группируются по одинаковым позициям. Вместо отдельных записей для каждой единицы товара, вы получаете одну запись с указанием общего количества, количества активных и списанных.

{

        {
            "name": "Вискас с курицей",
            "manufacturer": "Mars",
            "category_id": 1,
            "animal_type_id": 1,

    ]

3treats

3rodents

        "status": "all_movments",

                "action": "adding",

GET /select_movment/<str:action> 🔒 Требуется токен NEW
Получить движения по типу действия (adding, disposal)

  • /select_movment/disposal - только списания

{
    "data": {
        "status": "adding_movments",
        "data": [
            {
                "id": 1,
                "name": "Вискас с курицей",
                "manufacturer": "Mars",
                "p_id": 1,
                "action": "adding",
                "date_time": "2024-03-17T12:30:45.123Z",
                "description": null,
                "employee": "Иван Петров",
                "employee_email": "ivan@example.com"
            }
        ]
    }
}

POST /delete_employees 🔒 Требуется токен 👑 Только администратор
Удалить сотрудника (только для администратора, position_id = 1)
{
    "uid": 5
}

📋 Параметры:

Поле Тип Обязательность Описание
uid integer ✅ Да ID пользователя для удаления

✅ Успешный ответ (200):

{
    "data": "user not found"
}

❌ Ошибка доступа (200 с ошибкой):

{
    "data": "no permission acces"
}

Коды ошибок

Код Описание
200 Успешный запрос (даже при ошибках может возвращать 200 с полем "error")
201 Успешное создание
400 Неверные данные запроса
401 Не авторизован (нет токена)
403 Доступ запрещен (недостаточно прав)
404 Ресурс не найден
500 Внутренняя ошибка сервера

Важные примечания

  • 📦 1. Каждая запись товара = одна физическая единица
    Каждая запись в БД соответствует одной коробке, мешку и т.д. Нет поля Quantity.
  • 🔄 2. Статусы товаров:
    exists - активный (на складе)
    deleted - списан/удален
  • 📋 3. Действия движений:
    adding - добавление товара
    disposal - списание товара
  • 👔 4. Должности:
    1 - Administrator (полный доступ)
    2 - Warehouse_Operator (ограниченный доступ)
  • 🔑 5. Токены:
    Токен автоматически создается при регистрации/входе и удаляется при выходе
  • 🆕 6. Новые эндпоинты:
    Добавлена фильтрация товаров (просроченные/списанные), движений по типу и сотрудников по должности
  • 📊 7. Группировка товаров (НОВОЕ!):
    Все GET запросы для товаров теперь возвращают сгруппированные данные:
    - /select_all - возвращает total_quantity, active_quantity, deleted_quantity
    - /select_expired и /select_deleted - возвращают quantity
    Это позволяет видеть общее количество одинаковых позиций на складе.
